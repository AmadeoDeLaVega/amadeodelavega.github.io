This manuscript studies a problem that shows up quickly when using coding agents: generated code can look convincing, compile cleanly, and still be wrong. The paper proposes a verifier harness that combines multiple layers of checking, including syntax validation, structural analysis, and dynamic testing, to catch bad patches more reliably.

For undergraduate CS students, the project offers a concrete view of how software testing, program analysis, and LLM-based development fit together. It is less about asking whether an AI can write code, and more about designing the machinery needed to trust its output.
